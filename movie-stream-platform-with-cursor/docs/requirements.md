# 動画配信プラットフォーム 要件定義書

## 1. プロジェクト概要

Udemyのような月額制動画配信プラットフォームを構築する。講師が配信する動画をユーザーが閲覧できるWebアプリケーション。

> **※ 本バージョンはデモ版のため、認証・決済機能は実装しない（後回し）**

---

## 2. 基本方針

| 項目 | 方針 |
|------|------|
| 認証 | なし（誰でもアクセス可能） |
| 決済 | なし（全コンテンツ無料で視聴可能） |
| 動画配信 | YouTube埋め込みで実装 |

---

## 3. 機能要件

### 3.1 トップページ（コース一覧）

- **目的**: ユーザーが利用可能なコースを一覧で確認する
- **機能**:
  - 複数のコースをカード形式などで配置
  - 各コースのサムネイル・タイトル・概要などを表示
  - コースをクリックすると、そのコースの詳細ページへ遷移

### 3.2 コース詳細ページ

- **目的**: コースの内容を確認し、視聴を開始する
- **機能**:
  - コースの詳細情報を表示（タイトル、説明、講師情報など）
  - 「コースを見る」ボタンを配置
  - ボタンクリックで、動画視聴ページへ遷移

### 3.3 動画視聴ページ（Udemy風レイアウト）

- **目的**: セッション単位で動画を視聴する
- **機能**:
  - **セッション構成**: コース内の動画をセッション（章・レッスン）ごとに分割
  - **動画プレイヤー**: YouTube埋め込みで動画を再生
  - **セッション一覧**: サイドバーまたは一覧でセッション・動画のリストを表示
  - **連続視聴**: 1本の動画を見終わったら、次の動画へ進める
  - UdemyのようなUI/UX（左に動画、右または下にセッション一覧など）

---

## 4. 画面遷移

```
[トップページ（コース一覧）]
        │
        │ コースをクリック
        ▼
[コース詳細ページ]
        │
        │ 「コースを見る」ボタンをクリック
        ▼
[動画視聴ページ（セッション一覧 + YouTube埋め込み）]
```

---

## 5. データ構造（想定）

### コース（Course）

- コースID
- タイトル
- 説明
- サムネイルURL
- 講師名（任意）

### セッション（Session）

- セッションID
- コースID
- セッション名（例: 「第1章: はじめに」）
- 表示順序

### 動画（Video）

- 動画ID
- セッションID
- タイトル
- YouTube動画ID（埋め込み用）
- 表示順序

---

## 6. 技術的な補足

- **動画再生**: YouTube iframe API または 通常の埋め込み（`youtube.com/embed/{videoId}`）
- **認証・決済**: 本デモでは未実装
- **データ管理**: 静的JSON、ローカルストレージ、または簡易バックエンドで管理可能

## 7. 今後の拡張（スコープ外）

- ユーザー認証（ログイン・会員登録）
